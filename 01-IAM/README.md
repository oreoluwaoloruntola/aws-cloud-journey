# IAM User Creation

## Objective

Learn how to create and use an IAM user instead of the AWS root account.

## What I Learned

- What AWS IAM is
- Why root accounts should not be used for daily tasks
- How to create an IAM user
- How to assign permissions
- How to sign in as an IAM user

## Steps I Performed

1. Logged into the AWS Management Console.
2. Opened the IAM service.
3. Created a new IAM user.
4. Granted the necessary permissions.
5. Saved the sign-in URL.
6. Logged out of the root account.
7. Signed in using the IAM user.
8. Reset the temporary password.

## Challenges

- I initially found it confusing to switch from the root account to the IAM user.
- I wasn't sure what the Account ID and CSV file were used for.
- After following the steps, I successfully logged in as the IAM user.

## Key Takeaways

- Never use the root account for daily work.
- IAM improves security by allowing users with limited permissions.
- Always save the sign-in URL and keep credentials secure.
